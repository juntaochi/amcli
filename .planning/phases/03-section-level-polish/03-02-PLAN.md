  <action>
Per VISL-03 and VISL-04 and locked user decisions, replace the binary lyrics highlighting with a 3-tier graduated dimming system.

Current code to replace (lines 598-606):
```rust
for (i, line) in lyrics.lines.iter().enumerate() {
    let style = if i == current_index {
        Style::default()
            .fg(theme.primary)
            .add_modifier(Modifier::BOLD)
    } else {
        Style::default().fg(theme.dim)
    };
    lines.push(Line::from(Span::styled(&line.text, style)));
}
```

Replace with:
```rust
for (i, line) in lyrics.lines.iter().enumerate() {
    let distance = (i as isize - current_index as isize).unsigned_abs();
    let style = if i == current_index {
        // Tier 1: Current line -- accent color + bold (brightest)
        Style::default()
            .fg(theme.accent)
            .add_modifier(Modifier::BOLD)
    } else if distance <= 2 {
        // Tier 2: Near lines (+-1-2) -- normal readable text
        Style::default().fg(theme.primary)
    } else {
        // Tier 3: Far lines (beyond +-2) -- faded
        Style::default().fg(theme.dim)
    };
    lines.push(Line::from(Span::styled(&line.text, style)));
}
```

Key details:
- Cast to isize before subtraction to avoid unsigned underflow panic when i < current_index
- Use unsigned_abs() for the absolute distance
- Current line: theme.accent + BOLD (per user decision -- accent is brighter/warmer than primary in every theme)
- Near lines (distance <= 2): theme.primary (normal readable text)
- Far lines (distance > 2): theme.dim (faded, existing behavior)

Do NOT add Modifier::DIM to far lines -- terminal-dependent and unreliable with RGB colors per research.
Do NOT build a color interpolation helper -- 3 existing theme colors provide sufficient visual tiers.
Do NOT change the draw_lyrics function signature.
Do NOT change the scroll calculation or the "NO LYRICS AVAILABLE" fallback.
